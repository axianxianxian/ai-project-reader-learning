# 目标

这个项目服务于两个目标：

1. 三个月后面试时，有一个可以展示、可以讲清楚的 AI 工程项目。
2. 用一个小项目把 AI 框架学习落到真实工程链路里。

## 项目一句话

AI Project Reader 是一个技术项目阅读助手：输入项目资料，输出结构化阅读路径、demo 计划、风险提醒和下一步动作。

## 为什么适合我

它不要求一开始做很复杂的模型能力，但会逼我理解：

- 接口设计
- prompt 组织
- 模型调用边界
- tool / 内部知识
- 结构化输出
- 业务代码如何消费 AI 结果
- 测试怎么覆盖 AI 应用

## 作品集讲法

我不是只“调了一个模型接口”。

我做的是一个小型 AI 应用骨架：它把用户输入转成 prompt，通过 AI Gateway 生成结构化报告，再让业务代码消费这个报告。后续可以扩展成 Spring AI、tool calling、RAG 和 agent workflow。
