# 交付排期 v1

日期：2026-07-19

## 规划原则

这个项目按真实交付项目推进：

- 先定 PRD
- 再定最小 Demo
- 再排每天产出
- 每天必须有可见交付物
- 不用“学了很多”当产出

## 时间投入假设

默认每天投入：

- 工作日：1-2 小时
- 周末或集中日：3-5 小时

如果当天只有 45 分钟，也必须收敛成一个小交付。

## 今天产出

今天不继续乱写功能。

今天目标是完成项目定盘：

- PRD v1
- 未来效果 Demo
- 两周排期
- 风险表
- 项目重点

今天结束标准：

- 你能说清这个项目要交付什么
- 你能说清第一阶段不做什么
- 你能说清每天要往哪里推进

## 第一周目标

主题：把“本地项目热读四件套”做成最小可用。

### Day 1：定盘和对齐

产出：

- PRD v1
- 未来效果 Demo
- 排期和风险表

验收：

- 产品输入和输出固定
- 本周重点明确

### Day 2：模型重构

产出：

- 新增项目地图模型
- 新增核心链路模型
- 新增 Top 5 文件模型
- 调整阅读报告模型

验收：

- 后端结构化对象能表达四件套
- Mock 输出四件套样例

### Day 3：扫描器升级

产出：

- 识别 README
- 识别 pom/build/package
- 识别启动类
- 识别 Controller / Service / Config
- 输出扫描摘要

验收：

- 用当前项目扫描出关键文件
- Top 5 文件不再完全靠 AI 猜

### Day 4：项目地图和 Top 5

产出：

- `/analyze-local` 输出项目地图
- 输出 Top 5 文件
- 每个结论带 evidenceFiles

验收：

- 当前项目能生成可信项目地图
- Top 5 文件有路径、角色、理由、顺序

### Day 5：核心链路

产出：

- 生成核心链路结构
- 至少支持 Spring Boot Controller -> Service -> Gateway 这种链路
- 缺失上下文时返回 missingContext

验收：

- 当前项目能输出 Controller 到 AiGateway 的主链路

### Day 6：页面原型联动

产出：

- 页面展示四件套
- 有输入区、状态区、结果区、追问区
- 先接 mock 或静态 JSON

验收：

- 用户能看到未来产品形态

### Day 7：本周收口

产出：

- README 更新
- 一份真实样例输出
- 一篇作品集日志草稿

验收：

- 能演示本地项目路径 -> 四件套输出

## 每天最小工作单元

如果当天时间少，只做一个小闭环：

```text
读一个文件
-> 改一个模型或接口
-> 跑一次测试
-> 写一句复盘
```

不要开很多线。
