DeepFlow 是一个面向可观测性的基础设施项目

它的核心目标是自动采集应用、网络和基础设施的运行数据，再把这些数据组织成可查询、可分析、可追踪的可观测平台。阅读时不要一开始钻源码，先抓住采集、传输、存储、查询这条主线。

先知道它做什么 它帮助团队理解系统运行状态，不是普通业务应用。
再知道数据怎么流 数据从 agent 采集，进入 server，再被查询和展示。
最后看怎么运行 先跑官方 demo 或最小部署，再回头看源码。

怎么最快跑起来

  1. 先读 README 的 quick start，不要先看所有目录。
  2. 确认依赖：Docker、Kubernetes 或本地开发环境。
  3. 用官方脚本跑一个最小 demo。
  4. 打开控制台，看是否能看到采集到的数据。

端到端链路

1
Agent 采集数据 部署在节点上，采集网络、系统调用、应用和资源数据。
2
数据进入后端 Server 接收和处理采集数据，做清洗、聚合和存储。
3
查询和分析 用户通过 UI、API 或查询层查看链路、指标和事件。
4
定位问题 最终目标是回答“哪里慢、哪里断、谁影响了谁”。

第一次应该读哪些文件

  • README.md：确认项目定位、运行方式和官方推荐路径。
  • agent/：理解数据从哪里来。
  • server/：理解核心处理逻辑在哪里。
  • cli/：理解用户如何操作系统。
  • Makefile：找到构建、测试、启动命令。

今天先不要深挖什么

不要一上来研究所有 eBPF 实现细节，也不要把每个目录都打开看。先确认系统角色和端到端数据流，能讲清楚“数据从哪里来，到哪里去，用户怎么消费”，这就是第一轮阅读完成。
