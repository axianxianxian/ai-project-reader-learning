这个项目是做什么的

它是一个 AI 工程热读助手：读取项目的基础文件、目录结构和关键源码，然后输出一份适合新人上手的项目阅读地图。

工程理解 AI 应用 作品集项目

最短跑通路径

  1. 准备 Java 和 Maven。
  2. 启动 Spring Boot 应用。
  3. 调用本地项目阅读接口。
  4. 查看结构化热读报告。

主链路

1
用户输入项目路径和阅读目标。
2
Controller 接收请求并交给 Service。
3
Scanner 读取 README、依赖文件、目录和关键源码。
4
PromptBuilder 组织提示词，AI Gateway 生成结构化结果。
5
系统返回热读报告和推荐追问。

用户下一步该做什么

  • 先读 README 和 Controller，确认入口。
  • 再读 Service，理解系统怎么编排。
  • 最后读 PromptBuilder，理解 AI 能力怎么落地。
  • 不要第一天研究所有文件。

已知不确定点

当前报告来自静态扫描和 mock AI，暂时不能保证读懂复杂业务语义。第一版要记录输入输出样本，靠人工验收迭代。
